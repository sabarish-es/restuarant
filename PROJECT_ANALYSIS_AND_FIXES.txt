ISSUE #1: ORDER CREATION FAILURE IN CASHIER PAGE
================================================================================

PROBLEM:
When users tried to create an order in the cashier page, the system displayed:
"Failed to create order: Failed to create order"

ROOT CAUSE ANALYSIS:
The order creation was failing because of a database column mismatch:

Frontend (cashier/page.tsx) was sending:
{
  "items": [...],
  "tableId": ...,
  "orderType": "dine-in|takeaway",
  "paymentMethod": "cash|card|upi"
}

Backend (orderController.js) was using WRONG column names:
- Using: "order_status" column (not in database)
- Using: "user_id" column (should be "cashier_id")
- Using: "total_amount" column (should be "total", "subtotal", "tax")
- Not handling "order_type" field properly
- Not creating order_items table with correct schema

Database (database.sql) actual schema:
- Column: "cashier_id" (not "user_id")
- Column: "status" (not "order_status")
- Columns: "subtotal", "tax", "total" (not "total_amount")
- Column: "order_type" (required)
- order_items needs: order_id, menu_item_id, quantity, unit_price

SOLUTION APPLIED:

1. Fixed /backend/controllers/orderController.js - createOrder function:
   BEFORE:
   INSERT INTO orders (table_id, customer_id, user_id, order_status, total_amount, notes)
   
   AFTER:
   INSERT INTO orders (table_id, customer_id, cashier_id, order_type, status, subtotal, tax, total, notes)

2. Fixed calculation logic:
   - Separated subtotal and tax calculations
   - Correctly using: subtotal + tax = total
   - Proper decimal formatting to avoid database type errors

3. Fixed order_items insertion:
   BEFORE:
   (order_id, menu_item_id, quantity, unit_price, total_price)
   
   AFTER:
   (order_id, menu_item_id, quantity, unit_price)

4. Fixed printBill function:
   - Changed reference from order.total_amount to order.total
   - Added subtotal and tax display in bill
   - Fixed item total calculation using unit_price * quantity

TROUBLESHOOTING
================================================================================

If order creation still fails:

2. Verify MySQL is running: mysql --version
3. Check database exists: mysql -u root -p -e "SHOW DATABASES;" | grep restaurant
4. Check tables exist: npm run db:init (run again)
5. Verify .env file has correct credentials
6. Check NEXT_PUBLIC_API_URL is correct in .env.local
7. Clear browser cache (Ctrl+Shift+Delete)
8. Restart both servers
